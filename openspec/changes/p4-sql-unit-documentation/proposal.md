## Why

Phases 1-3 deliver a complete, tested SQL Unit framework. Phase 4 focuses on documentation - making the tool accessible to users. Comprehensive documentation includes:
- Getting started guide for new users
- Detailed feature documentation
- API reference
- Examples and tutorials
- Best practices
- Troubleshooting

This documentation site (mkdocs) makes sql-unit discoverable and usable for the community.

## What Changes

- **Documentation site** - mkdocs-based documentation with multiple sections
- **User guide** - Getting started, basic usage, advanced features
- **API reference** - Auto-generated from docstrings
- **Examples** - Real-world examples and tutorials
- **Database guides** - Setup instructions for each supported database
- **Troubleshooting** - Common issues and solutions

## Capabilities

### New Capabilities

- `documentation-site`: Comprehensive mkdocs documentation site with user guide, API reference, examples, and troubleshooting

## Impact

- **User onboarding** - Clear path for new users to get started
- **Feature discovery** - Users learn about capabilities through docs
- **Community building** - Professional documentation attracts contributors
- **Support reduction** - Common questions answered in docs
