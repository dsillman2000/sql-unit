## Why

Phases 1-4 deliver a complete, tested, documented sql-unit project. Phase 4's Release focuses on preparing the project for public distribution via PyPI and GitHub. This phase includes:
- Version management and release notes
- Package metadata and publishing
- Release automation
- Public announcement and promotion

## What Changes

- **Version tagging** - Semantic versioning and git tags
- **Release notes** - Changelog and release documentation
- **PyPI publishing** - Package published to Python Package Index
- **Release checklist** - Quality gates before release
- **Announcement** - Social media, communities, etc.

## Capabilities

### New Capabilities

- `release-management`: Release preparation, PyPI publishing, and public announcement

## Impact

- **Public availability** - Users can install sql-unit via pip
- **Version management** - Clear versioning and upgrade path
- **Community visibility** - Announcement and promotion
- **Maintenance clarity** - Release notes document changes
